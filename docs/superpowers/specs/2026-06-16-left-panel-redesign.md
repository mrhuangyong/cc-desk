# cc-desk 左栏重设计 — 设计文档

- **日期**：2026-06-16
- **阶段**：增量功能（基于已交付原型）
- **状态**：待用户审阅

## 1. 目标布局

左栏（会话树态）从上到下：

```
┌──────────────────────────────┐
│ 顶部功能区（三按钮一行）        │
│ [➕ 新建会话] [🔍 搜索] [⚡ 技能] │
├──────────────────────────────┤
│ 工作区工具条                   │
│ 工作区   [⇕ 展开/折叠] [↕ 排序筛选] │
│ [🔍 搜索过滤框 ───────────]    │
├──────────────────────────────┤
│ 项目会话树（可展开/折叠、过滤）  │
│   📁 项目A                    │
│     💬 会话1                  │
│     💬 会话2                  │
│   📁 项目B                    │
│     💬 会话1                  │
└──────────────────────────────┘
```

## 2. 顶部功能区

三个按钮，等宽排列：

| 按钮 | 行为 |
|------|------|
| ➕ 新建会话 | 在**当前激活会话所属项目**下新建会话（沿用空会话去重逻辑：该项目已有空会话则切换过去）。无激活会话时取第一个项目兜底 |
| 🔍 搜索 | 打开**全局搜索弹窗**（见 §4），可搜会话、命令等 |
| ⚡ 技能 | 打开**技能列表面板**（见 §5），显示本地已安装技能（原型用 mock） |

## 3. 工作区工具条

对下方项目会话树的操作：

- **标题"工作区"**
- **⇕ 展开/折叠**：一键展开/收起所有项目（切换全局展开态）
- **↕ 排序/筛选**：弹出小菜单（原型占位，含"按时间""按名称"等选项，暂不实装排序逻辑）
- **搜索过滤框**：实时过滤下方会话树——按会话标题匹配，输入即时过滤；空则显示全部

## 4. 全局搜索弹窗（顶部"搜索"触发）

- 居中浮层（模态），带输入框。
- 输入关键词后列出匹配结果，分两组：
  - **会话**：标题匹配的会话（点击跳转：选中该会话 + 关闭弹窗）
  - **命令**：一组 mock 命令（如"新建项目""切换主题""打开设置"——原型占位，点击关闭弹窗）
- ESC 或点遮罩关闭。

## 5. 技能列表面板（顶部"技能"触发）

- 居中或侧浮层面板。
- 列出 mock 的本地技能（如"代码审查""生成测试""重构建议"——占位名称 + 简短描述）。
- 原型阶段：纯展示，点击技能暂只关闭面板（后续可接"把技能作为消息发送"）。

## 6. 项目会话树（重构现有 ProjectTree）

- **项目默认全部展开**（会话可见）。
- 项目行点击：展开/收起该项目的会话。
- 项目行 hover 操作：📂 文件树 + 🗑️ 删除（**移除原有的 ➕ 新建会话**，新建统一走顶部按钮）。
- 会话行：点击选中，hover 显示 🗑️ 删除（不变）。
- 工作区过滤框有值时：只显示标题匹配的会话（项目无匹配会话则整个项目隐藏）。

## 7. 数据/状态变更

- **新增 UI 状态**（放 LeftPanel 本地 state，不必进 store）：
  - `expandedProjects: Set<string>`（展开的项目 id，默认全部）
  - `treeFilter: string`（工作区过滤框文本）
  - 三个面板的开关：`searchOpen` / `skillsOpen`
- **复用现有**：ADD_SESSION（顶部新建会话时，先解析当前项目 id）、SELECT_SESSION（搜索弹窗跳转）。
- "当前激活会话所属项目"由 `activeSessionId` 反查得到（已有逻辑）。
