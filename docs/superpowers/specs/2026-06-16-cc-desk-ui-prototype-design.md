# cc-desk 桌面应用 — UI 原型设计文档

- **日期**：2026-06-16
- **阶段**：UI 原型（第一轮交付物）
- **技术栈**：Electron + React + TypeScript + Vite（UI 原型阶段以可视化界面为交付目标）
- **状态**：待用户审阅

## 1. 概述

### 1.1 产品定位

cc-desk 是一个**类 Claude Code Desktop 的 AI 工作台桌面应用**。核心形态是**工作台（C 型）**：以 AI 对话为主角，辅以项目/会话管理、文件树、终端，并集成一个**完整的内置浏览器**（可打开任意网址、地址栏、前进后退、多标签）。

### 1.2 本轮目标

本轮交付一个**可视化的 UI 原型**，验证以下设计方向是否正确：

- 三栏布局骨架与各区域职责
- 左栏（项目→会话）的完整交互规格
- 右栏多 Tab 面板的行为规则
- 多主题系统的架构与四种内置主题的视觉

原型为静态/半交互式界面（HTML+CSS+少量 JS），不含真实的 AI 对话、文件系统读写、浏览器内核集成。这些在后续实现计划中展开。

## 2. 整体布局

**三栏结构 + 自定义标题栏**，从左到右：

```
┌──────────────────────────────────────────────────────────────┐
│  自定义 TitleBar  [traffic lights]   项目名   [主题切换器] [⚙]  │
├────────────┬──────────────────────────┬──────────────────────┤
│            │                          │                      │
│  左栏       │        中栏               │      右栏             │
│  240px     │   对话区（flex:1）         │   280px              │
│            │                          │   可拖拽缩放           │
│  项目→会话   │   消息流 + 输入框          │   多 Tab 面板         │
│  /文件树    │                          │                      │
│            │                          │                      │
└────────────┴──────────────────────────┴──────────────────────┘
```

- **左栏**：固定 240px，承载"项目→会话"树与（切换态下）项目文件树。
- **中栏**：`flex:1`，AI 对话主区，永远占据最大面积。
- **右栏**：默认 280px，可拖拽缩放，承载多 Tab 面板。
- **TitleBar**：自定义实现（`frame: false` + 自绘标题栏），左侧 macOS 红绿灯按钮，中间项目名，右侧快捷主题切换器与设置入口。

## 3. 左栏交互规格

左栏只有两个状态：**会话树态**（默认）与**文件树态**（点击项目文件树图标进入）。

### 3.1 会话树态（默认）

两级的扁平列表（非分叉树）：

```
📁 项目A
   💬 会话1
   💬 会话2
📁 项目B
   💬 会话1
```

#### 3.1.1 项目行（hover）

鼠标移入项目行，右侧依次显示三个操作图标：

- **📂 项目文件树**：点击 → 左栏整体切换为文件树态（见 3.2）
- **➕ 新增会话**：点击 → 为当前项目新增一条会话（见 3.1.3 去重规则）
- **🗑️ 删除项目**：点击 → 进入删除二次确认（见 3.1.4）

#### 3.1.2 会话行（hover）

鼠标移入会话行，右侧只显示一个操作图标：

- **🗑️ 删除会话**：点击 → 进入删除二次确认（见 3.1.4）

#### 3.1.3 新增会话去重规则

点击 ➕ 时：

- 若当前项目下**已存在空会话**（消息数为 0，含"发过消息又清空"的情况）→ **不新建**，直接**切换到**那条空会话。
- 若不存在空会话 → 新建一条空会话并切换过去。

#### 3.1.4 删除二次确认（项目与会话共用同一套）

删除是危险操作，采用**就地二次确认**，流程：

1. hover 行 → 显示 `🗑️` 删除图标
2. 点击 `🗑️` → 图标变为 `✅` 确认图标（行可高亮警示色）
3. **点击 `✅`** → 执行删除
4. **不点击 ✅ 而是移开鼠标 / 点击别处** → 图标还原为 `🗑️`（取消删除）

**级联删除**：删除项目时，其下所有会话一并删除。

### 3.2 文件树态

点击项目行的 📂 进入。**整个左栏**变为该项目的文件列表：

```
← 项目A          （顶部返回按钮）
📁 src
   main.tsx
   App.tsx
   📁 components
📁 public
   package.json
```

- **顶部返回按钮**（`← 项目名`）：点击 → 回到会话树态（保留之前的位置/状态）。
- **点击某个文件** → 在**右栏新开一个文件 Tab**（见 4.3 去重规则）。

## 4. 右栏多 Tab 面板规格

右栏是绑定到**当前会话**的多 Tab 容器。

### 4.1 Tab 类型（本期三种）

- **文件预览/编辑**：展示选中的文件内容。
- **浏览器**：内置浏览器，地址栏 + 前进后退 + 网页渲染。
- **终端**：命令行终端。

（暂无其他类型；后续可扩展。）

### 4.2 Tab 与会话的绑定关系

- 右栏 Tab **绑定当前会话**：切换会话 → 右栏换一组 Tab。
- 每个会话维护自己的一组 Tab 状态。

### 4.3 多开与去重规则

- **同类型可开多个**：可以有多个浏览器 Tab（不同网址）、多个终端 Tab。
- **同一文件只允许一个 Tab**：打开已存在于右栏的文件 → **不新开**，而是**切换到**那个已存在的文件 Tab。

### 4.4 Tab 的增删

- **新增**：Tab 栏末尾有 `+` 按钮，点击后选择 Tab 类型；或通过左栏点文件、对话中的"打开文件"等行为触发。
- **删除**：每个 Tab 有关闭按钮（`×`），点击即关。

### 4.5 空 Tab 状态

关掉最后一个 Tab → 右栏显示**空白占位**（提示"暂无打开的面板"之类），右栏**不自动收起**。

## 5. 主题系统规格

### 5.1 内置主题（四种）

| ID | 名称 | 底色 | 强调色 | 字体 | 圆角 |
|----|------|------|--------|------|------|
| `dark-warm` | 暖色暗夜 | 深棕黑 `#1a1714` | 烧橙 `#d97757` | 无衬线 | 中等圆角 |
| `dark-cool` | 冷峻深空 | 冷灰黑 `#0d1117` | 电子蓝 `#2f81f7` | 无衬线 | 中等圆角 |
| `light-editorial` | 纸感明亮 | 米白 `#faf8f3` | 墨棕 `#8b6f47` | 衬线 | 中等圆角 |
| `dark-acid` | 酸性极客 | 纯黑 `#0a0a0a` | 荧光酸绿 `#ccff00` | 等宽 | 直角（无圆角） |

### 5.2 架构要求

- **所有颜色、圆角、字体走 CSS 变量**（`:root` / `[data-theme="xxx"]`），切换主题只改变量值，不改组件结构。
- 每个主题是一组变量值的集合，组件代码不感知具体主题。

### 5.3 切换行为

- **入口**：TitleBar 右侧快捷主题切换器（图标点开下拉，列出四种主题，当前主题高亮）。
- **实时生效**：点击即换色，无"应用/保存"步骤。
- **本地持久化**：用户选择的主题存本地（localStorage / Electron store），下次打开应用恢复上次主题。
- **默认主题**：`dark-warm`（首次启动无记录时）。

## 6. 组件划分（原型层面）

按"单一职责 + 清晰边界"拆分，每个组件可独立理解和预览：

| 组件 | 职责 | 依赖 |
|------|------|------|
| `AppShell` | 三栏 + TitleBar 的整体骨架与布局 | 各栏组件 |
| `TitleBar` | 自定义标题栏、红绿灯、项目名、主题切换器、设置入口 | 主题 Context |
| `LeftPanel` | 左栏容器，管理"会话树态 / 文件树态"切换 | ProjectTree / FileTree |
| `ProjectTree` | 项目→会话列表、hover 操作、删除二次确认、新增去重 | 会话状态 store |
| `FileTree` | 项目文件列表、返回按钮 | 文件数据（原型用 mock） |
| `DeleteConfirmIcon` | 封装"🗑️→✅"的就地二次确认交互 | 无 |
| `ChatArea` | 对话消息流 + 输入框 | 会话消息 store |
| `RightPanel` | 多 Tab 容器、Tab 增删、空白占位 | TabBar / 各 Tab 内容 |
| `TabBar` | Tab 标签栏、关闭按钮、新增按钮、去重逻辑 | Tab 状态 store |
| `FileTab` / `BrowserTab` / `TerminalTab` | 三种 Tab 内容 | 各自 mock 数据 |
| `ThemeProvider` + `useTheme` | 主题 Context，提供当前主题与切换方法、持久化 | localStorage |

## 7. 数据与状态（原型层面，均为 mock）

原型不接真实数据，用内存 mock 状态演示交互：

- **项目/会话**：mock 的项目列表与会话列表，支持增删（验证交互流程）。
- **文件**：mock 的文件树结构，点开显示 mock 文件内容。
- **消息**：mock 的对话消息，可演示发送（回显到消息流）。
- **浏览器/终端**：浏览器用 iframe 演示网页渲染，终端用静态文本演示。

（真实的数据持久化、文件系统、AI 对接、浏览器内核集成在后续实现计划中设计。）

## 8. 原型验证清单（本轮"完成"的标准）

原型交付后，以下交互可在浏览器中实际操作并验证：

- [ ] 三栏布局正确呈现，比例与视觉符合设计
- [ ] 四种主题可通过 TitleBar 切换器实时切换并持久化
- [ ] 左栏：项目/会话 hover 显示对应操作图标
- [ ] 左栏：删除二次确认流程（🗑️→✅→执行 / 还原）正确
- [ ] 左栏：新增会话的去重规则正确（已有空会话则切换）
- [ ] 左栏：点 📂 进入文件树态，点 ← 返回会话树态
- [ ] 左栏：点文件在右栏新开文件 Tab，重复打开同文件切换不新开
- [ ] 右栏：Tab 增删正确，关掉最后一个显示空白占位
- [ ] 右栏：切换会话时右栏 Tab 组跟着切换

## 9. 不在本轮范围（明确排除）

- 真实 AI 对话能力（接 API、流式输出、工具调用）
- 真实文件系统读写
- 真实终端进程
- 内置浏览器内核的完整集成（原型用 iframe 演示）
- 数据持久化到磁盘（原型用内存 + localStorage 演示主题）
- 可拖拽缩放栏宽度的精确实现（原型用固定宽度，预留接口）
- 设置页的完整内容
- 应用打包与分发

这些在后续 spec / 实现计划中单独设计。
